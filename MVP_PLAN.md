# 설스터디 MVP 화면 구성 기획안

## 서비스 개요

- **서비스명**: 설스터디
- **설명**: 자체 콘텐츠 기반 수능 국영수 학습코칭 플랫폼
- **핵심 가치**: 노션보다 편리하게 학습 코칭을 받을 수 있는 웹 서비스
- **테스트 기준**: 멘토 1명 + 멘티 2명 (사전 세팅 계정, 회원가입 없음)
- **매칭 구조**: 1명의 멘토 → 여러 멘티 담당 (1:N), 멘티는 1명의 멘토와만 매칭, 한 멘토가 국/영/수 전 과목 피드백

---

## 역할별 환경

| 역할 | 환경 | 비고 |
|------|------|------|
| 멘토 | 웹(PC) 중심 | 반응형 구현 시 가산점 |
| 멘티 | 웹앱(모바일) 중심 | 모바일 퍼스트 |

## 노션 현행 구조 분석

현재 노션으로 운영 중인 멘티 페이지(민유진)의 실제 구조를 분석한 결과:

```
📄 민유진 (멘티 메인 페이지)
├── 📋 [오늘 할 일] — DB 테이블 (필터: 오늘 날짜)
│   └── 스키마: 이름, 날짜, 과목(롤업), 목표(릴레이션), 학습지(롤업), 멘토확인(체크박스)
│
├── 📋 [어제자 피드백 보기] — 같은 DB, 다른 뷰 (필터: 어제 날짜)
│   └── 피드백 작성 완료된 항목만 표시
│
├── 3칼럼 바로가기 영역
│   ├── 📅 월간계획표 → 하위 페이지
│   ├── ❓ 질의응답
│   └── 📝 서울대쌤 칼럼 → 하위 페이지
│
├── 2칼럼 기록 영역
│   ├── 주간 학습리포트 → 1주차, 2주차, 3주차, 4주차 하위 페이지
│   └── 줌 미팅 피드백 → 민유진 1st, 2nd 하위 페이지
│
└── 📋 약점 맞춤 솔루션 — DB 테이블
    └── 스키마: 보완점, 과목(국/영/수 select), 추가자료(파일)
```

### 핵심 데이터 모델 (노션 DB 기준)

**DB1: 오늘 할 일**
| 필드명 | 타입 | 설명 |
|--------|------|------|
| 이름 | title | 할 일 제목 |
| 날짜 | date | 수행 날짜 |
| 과목 | rollup (select) | 국어/영어/수학 (목표 릴레이션에서 롤업) |
| 목표 | relation | 약점 맞춤 솔루션 DB와 연결 |
| 학습지 | rollup (file) | 목표에 연결된 학습지 파일 |
| 멘토 확인 | checkbox | 멘토가 제출물 확인 여부 |
| 오늘 | formula | 오늘 날짜 필터용 |
| 어제 | formula | 어제 날짜 필터용 |

**DB2: 약점 맞춤 솔루션**
| 필드명 | 타입 | 설명 |
|--------|------|------|
| 보완점 | title | 보완할 학습 내용 |
| 과목 | select | 국어/영어/수학 |
| 추가자료 | file | PDF 학습지 등 첨부 파일 |

## 1. 공통 — 로그인 페이지

### 화면: `/login`

- 설스터디 로고
- 아이디 / 비밀번호 입력
- 로그인 버튼
- 역할(멘토/멘티)에 따라 로그인 후 각각의 홈으로 라우팅

> 회원가입 페이지 불필요. 테스트 계정 사전 세팅.

## 2. 멘티 화면 (모바일 웹앱)

### 2-1. 일일 플래너 (홈) `/mentee/planner`

멘티의 메인 화면. 노션의 멘티 페이지를 모바일에 최적화한 구조.

**상단 영역**
- 날짜 이동 버튼 (< 2025.01.27 월 >)
- 주 단위 미니 캘린더 (해당 주차의 모든 할 일 목록을 요약해서 보여주는 뷰, 선택된 날짜 하이라이트)
- 월간 캘린더 전환 토글 (탭하면 월간 달력 오버레이)

**코멘트/질문 영역**
- 코멘트·질문 입력란 (오늘의 각오, 멘토 질문, 회고 등 다목적 사용)

**[오늘 할 일] 섹션**
- 해당 날짜의 할 일 리스트 (노션 DB 테이블 뷰 대응)
  - 할 일 이름 + 과목 태그 (국어🔴 / 영어🔵 / 수학🟢)
  - 공부 시간 기록 입력 (완료 후 몇시~몇시 직접 입력 또는 실시간 기록 버튼)
  - 멘토 고정 할 일 → 수정/삭제 불가, 별도 표시, 인증 사진 업로드 **필수**
  - 멘티 추가 할 일 → [+ 할 일 추가] 버튼, 인증 사진 업로드 **선택**
- 할 일 항목 클릭 → 과제 상세 페이지로 이동
- **미래 할 일 앞당겨 완료 불가** (해당 날짜에만 완료 처리 가능)

**[플래너 마감] 버튼**
- 모든 할 일을 마친 후 멘티가 직접 [플래너 마감/피드백 요청] 버튼을 눌러야 멘토에게 넘어가는 **수동 완료 방식**

**[어제자 피드백 보기] 섹션**
- 어제 날짜 기준, 멘토 피드백이 등록된 할 일만 표시
- 할 일 이름 + 과목 + 피드백 요약 미리보기
- 클릭 시 피드백 상세 확인

**Zoom 미팅 신청 버튼**
- **플로팅 버튼** (화면 우측 하단, 하단 탭 바 위에 고정)
- 탭 시 간단한 신청 폼 (바텀시트 또는 모달)
  - 희망 날짜 (date picker)
  - 희망 시간 (time picker)
  - 신청 버튼
- 신청 후 상태: 대기중 → 확인됨
- 별도 페이지 없이 모달로 처리 (최소 기능)

**하단 네비게이션 바 (탭 3개)**
- 플래너(홈) | 학습자료 | 마이페이지

---

### 2-2. 과제 상세 페이지 `/mentee/task/:id`

할 일 클릭 시 진입. 노션에서 할 일 행을 열었을 때의 상세 뷰 대응.

- 과제명 / 과목 태그 / 목표 (릴레이션된 보완점 표시)
- **탭 형태 UI**로 학습지 영역과 업로드 영역 구분
  - **[학습지] 탭**: 학습지 파일 (PDF 다운로드 버튼)
  - **[제출] 탭**: 공부 인증 사진 업로드 (카메라 촬영 또는 갤러리에서 JPG 선택, 썸네일 미리보기)
- **업로드 정책**:
  - 멘토 생성 과제 → 인증 사진 업로드 **필수** (업로드 없이 완료 불가)
  - 멘티 생성 과제 → 인증 사진 업로드 **선택**
- 날짜가 지나도 이전 과제 업로드 가능
- 멘토 확인 상태 표시 (체크 여부)

---

### 2-3. 피드백 확인 `/mentee/feedback/:date`

노션의 [어제자 피드백 보기] 테이블 뷰를 상세 화면으로 확장.

- 날짜 선택 (상단 날짜 이동)
- 해당 날짜의 피드백 리스트 (할 일별)
  - 할 일 이름 + 과목 태그
  - **요약 영역**: 멘토가 중요 표시한 핵심 내용 강조
  - **상세 펼침**: 할 일별 상세 피드백 텍스트
- **총평 영역**: 해당 날짜 전체 총평

---

### 2-4. 학습자료 `/mentee/resources`

노션의 3칼럼 바로가기(월간계획표, 서울대쌤 칼럼)와 약점 맞춤 솔루션 DB를 통합.

- **월간계획표** 카드 → 월간 일정 조회 (읽기 전용)
- **서울대쌤 칼럼** 카드 → 칼럼 목록 페이지로 이동
- **약점 맞춤 솔루션** 리스트
  - 과목별 필터 (전체/국어/영어/수학)
  - 보완점 + 추가자료 파일 다운로드

---

### 2-5. 마이페이지 `/mentee/mypage`

> 핵심 기능이 아님. 간단한 정보만 표시하면 충분.

- **프로필 영역**: 프로필 이미지, 닉네임
- **월간 리포트**: 월 단위 리포트 조회 (노션의 학습리포트 대응)

## 3. 멘토 화면 (PC 웹)

### 3-1. 담당 멘티 목록 (홈) `/mentor/students`

**상단 헤더**
- 알림 아이콘 (🔔) + 미확인 건수 배지 + **텍스트 라벨 (예: "알림 1건")**
  - 아이콘만으로는 인지가 어려우므로 라벨 추가
  - 클릭 시 알림 드롭다운: Zoom 미팅 신청 내역, 플래너 마감 알림 등
  - Zoom 신청 항목: 멘티명, 희망 일시, [확인] 버튼
  - 확인 시 상태가 "확인됨"으로 변경 (멘티 측에도 반영)

- 담당 멘티 카드 리스트
  - 멘티 이름 / 프로필
  - 오늘 할 일 현황 (n/m 완료)
  - 멘토 확인 대기 건수
  - 최근 피드백 작성 여부
- 멘티 카드 클릭 → 해당 멘티 상세 관리 화면

---

### 3-2. 멘티 상세 관리 `/mentor/students/:id`

노션의 멘티 페이지를 멘토 관점에서 재구성. 한 화면에서 할 일 관리 + 피드백 작성.

**상단**
- 멘티 이름 + 날짜 이동 (< 날짜 >)
- 주간/월간 캘린더 전환

**좌측 패널 — 할 일 & 제출 현황**
- 해당 날짜의 할 일 테이블 (노션 DB 테이블 뷰와 동일 구조)
  - 이름 | 과목 | 목표 | 학습지 | 멘토 확인
- 제출된 인증 사진 미리보기 (클릭 시 확대)
- 멘토 확인 체크박스 토글

**우측 패널 — 피드백 작성**
- 선택한 할 일에 대한 피드백 입력
- 중요 표시 체크 (요약에 노출)
- 총평 입력란
- 저장 버튼

---

### 3-3. 할 일 등록 `/mentor/students/:id/tasks/new`

멘티에게 할 일 부여. 노션에서 DB 행을 추가하는 것과 동일.

- 날짜 선택
- 할 일 이름 입력
- 목표 선택 (약점 맞춤 솔루션 DB에서 선택 → relation)
  - 과목은 목표 선택 시 자동 매핑 (rollup)
- 학습지 업로드 (설스터디 칼럼 or PDF 파일)
- 저장 버튼

---

### 3-4. 약점 맞춤 솔루션 관리 `/mentor/students/:id/solutions`

노션의 약점 맞춤 솔루션 DB 관리.

- 보완점 목록 (테이블 형태)
  - 보완점 | 과목 | 추가자료
- 항목 추가/수정
- 할 일 등록 시 여기서 목표를 참조

## 4. 화면 흐름 요약

```
[로그인]
  ├─ 멘티 로그인 (모바일 웹앱)
  │   ├─ 🏠 일일 플래너 (홈)
  │   │   ├─ [오늘 할 일] 리스트
  │   │   │   └─ 과제 상세 (할 일 클릭)
  │   │   ├─ [어제자 피드백 보기]
  │   │   │   └─ 피드백 상세
  │   │   └─ [Zoom 미팅 신청] (모달)
  │   ├─ 📚 학습자료
  │   │   ├─ 월간계획표
  │   │   ├─ 서울대쌤 칼럼
  │   │   └─ 약점 맞춤 솔루션
  │   └─ 👤 마이페이지
  │       └─ 월간 리포트
  │
  └─ 멘토 로그인 (PC 웹)
      └─ 🏠 담당 멘티 목록 (홈)
          ├─ 🔔 알림 (Zoom 신청 확인, 플래너 마감 등)
          └─ 멘티 상세 관리
              ├─ 할 일 등록
              ├─ 피드백 작성 (할 일별)
              └─ 약점 맞춤 솔루션 관리
```

## 5. 노션 → 웹 전환 매핑

| 노션 현행 구조 | MVP 웹 화면 | 비고 |
|---------------|-------------|------|
| 오늘 할 일 DB (테이블 뷰, 오늘 필터) | 멘티 플래너 할 일 리스트 | 날짜 필터는 formula 대신 쿼리로 |
| 어제자 피드백 보기 (테이블 뷰, 어제 필터) | 멘티 플래너 하단 피드백 섹션 | 같은 DB, 뷰만 다름 |
| 3칼럼 바로가기 (월간계획표/질의응답/칼럼) | 학습자료 탭 | 모바일에서는 세로 카드로 |
| 학습리포트 (하위 페이지) | 마이페이지 내 월간 리포트 | 읽기 전용, 월 단위 발송 |
| 약점 맞춤 솔루션 DB | 학습자료 탭 + 멘토 관리 화면 | 멘토만 CRUD |
| 할 일 행 상세 (노션 페이지 열기) | 과제 상세 페이지 | 학습지+인증사진 |
| 멘토 확인 체크박스 | 멘토 상세관리 패널 | 제출 확인용 |

## 6. MVP 범위 정리

### 필수 (Must Have)
- [ ] 로그인 (멘토/멘티 역할별 계정, 테스트 계정 3개)
- [ ] 멘티 - 일일 플래너 (날짜 이동, 주간 할 일 요약 뷰, 월간 캘린더, 오늘 할 일, 공부 시간 기록)
- [ ] 멘티 - 플래너 마감/피드백 요청 버튼 (수동 완료 방식)
- [ ] 멘티 - 어제자 피드백 보기 (플래너 하단)
- [ ] 멘티 - 과제 상세 (학습지 탭/제출 탭, 업로드 필수/선택 정책 적용)
- [ ] 멘티 - 과목별 피드백 확인 (요약 + 상세)
- [ ] 멘토 - 담당 멘티 목록
- [ ] 멘토 - 할 일 등록 (이름, 날짜, 목표 연결, 학습지 업로드)
- [ ] 멘토 - 피드백 작성 (할 일별 + 중요 표시 + 총평)
- [ ] 멘토 - 멘토 확인 체크
- [ ] 멘티 - Zoom 미팅 신청 (날짜, 시간 선택 → 모달)
- [ ] 멘토 - 알림 아이콘 + Zoom 신청 확인 (드롭다운)
